# Time To Value

Time to value is where the reader gets the first useful answer, not where the first CTA appears.

| Page type | First value should appear by | What counts as value |
|---|---:|---|
| Definition | First 10% | Clear definition and when it matters |
| How-to | First 15% | Steps or immediate path to completion |
| Template | Immediately | The template, example, or copyable artifact |
| Alternatives/listicle | Before first CTA | Criteria and first valid options |
| Comparison | Above the fold | Decision criteria and summary recommendation |
| Commercial category | First 20% | Shortlist logic, use cases, and buying criteria |
| Troubleshooting | First 10% | Likely causes and fixes |
| Thought leadership | First 15% | Clear thesis and stakes |

## Failure Patterns

- Long brand intro before answering.
- CTA or product walkthrough before the promised value.
- Generic definitions delaying a template or comparison.
- Inflated list length that delays useful recommendations.
- "What is X" content on a query where the reader wants choices.

## Planning Requirement

The page brief should state exactly where first value appears:

`First value: [section/asset/answer] must appear before [CTA/product section/background].`
