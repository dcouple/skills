# Good Vs Bad Examples

Use these examples as patterns, not as brand gossip.

## Listicles

Good pattern:
- Defines selection criteria.
- Includes only tools that satisfy the query.
- Gives each tool comparable treatment.
- States specific tradeoffs.
- Places CTAs after useful recommendations.

Bad pattern:
- Pads the list to create space for the brand.
- Ranks the brand first without evidence.
- Gives the brand several times more space than competitors.
- Includes tools outside the expected answer set.
- Uses vague self-limitations and sharper competitor criticisms.

## Comparisons

Good pattern:
- Applies the same criteria to every option.
- Names where each option wins and loses.
- Uses firsthand observations, screenshots, pricing, or methodology.
- Lets the reader choose based on use case.

Bad pattern:
- Pretends to be neutral while routing every criterion to the brand.
- Uses unsupported competitor claims.
- Hides the brand's real limitations.
- Turns the page into a disguised landing page.

## Product Promotion

Good pattern:
- Product appears where it helps the reader finish the job.
- CTA follows value.
- Product copy is shorter than or equal to the editorial answer it supports.

Bad pattern:
- Product angle changes the answer.
- Reader must pass multiple CTAs before getting value.
- The page stops being useful if CTAs are removed.

## Calibration Rule

Ask: "Would this recommendation still make sense if every brand name were hidden?"
