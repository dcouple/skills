# Page Strategy Workflow

Use this reference to turn a page idea into a brief.

## Brief Structure

1. Page thesis: one sentence describing why this page should exist.
2. Target query: primary query, secondary queries, and query class.
3. Searcher job: what the reader came to accomplish.
4. Page type: how-to, comparison, alternatives, template, landing page, glossary, guide, checklist, or tool page.
5. Success criteria: what the reader should be able to decide or do after reading.
6. Product role: lead, contextual, related, light mention, or excluded.
7. Proof burden: what must be shown for the page to be trusted.
8. Time to value: where the first real answer must appear.
9. Structure: section-by-section outline.
10. Differentiation: what this page does better than current winners.
11. Exclusions: what not to include because it would distort intent or add noise.
12. Review risks: what `$page-review` should check after drafting.

## Decision Rules

- If the query expects a decision, give criteria before selling.
- If the query expects examples or templates, show usable examples before context.
- If the product is not a native answer, do not make it the frame of the page.
- If the expected answer set excludes the product, the product can appear only as a related workflow or CTA after value.
- If firsthand proof is unavailable for a comparison/listicle, recommend narrower scope or a research step before writing.

## Strategy Questions

- What would make a searcher stop searching?
- What would feel like bait?
- What should appear before the first CTA?
- What claims require screenshots, data, tests, quotes, or methodology?
- Would the page still be useful if the product were removed?
- Does the outline answer the query, or route the query into the funnel?
